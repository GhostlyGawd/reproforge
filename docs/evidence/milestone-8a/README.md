## Local and live proof boundary

PGlite runs PostgreSQL in WebAssembly and closes deterministic local SQL
semantics for migrations, constraints, transactions, rollback, and properties.
The live gate separately applied all six migrations to Neon, recorded their
checksums, collapsed eight concurrent idempotent starts to one record, denied a
foreign-tenant read, and recovered one expired lease exactly once.

The Neon adapter uses the interactive-transaction `Pool` path and is created
lazily so a credential-free build performs no connection. The restore drill
used isolated schemas over the provisioned unpooled URL because Neon correctly
rejects custom `search_path` startup parameters on its pooled endpoint; the
application path remains pooled.

The Vercel Blob adapter requests private immutable objects, uses cache-bypassed
reads, returns no provider URL, and supports platform OIDC or a local
read-write token. The live store denied a direct unauthenticated URL, returned
the exact bytes through the authorized adapter, and denied access after
deletion.

The Vercel Queues adapter uses identifier-only messages, the durable event ID
as its provider idempotency key, bounded retention, the SDK callback contract,
and application-owned poison retry limits. The live service accepted the exact
six-field message and returned a provider message identity. Duplicate consumer
delivery and expired-lease recovery were then proven against live Postgres,
which remains authoritative. A deployed Vercel callback/redelivery observation
is intentionally deferred to hosted integration evidence and is not claimed
here.

Quota, cancellation, retention scheduling, deletion-claim recovery, and
append-only tombstone behavior are proven against PostgreSQL semantics. Live
provider tests used the same private-object deletion path and left zero active
test tenants, cases, artifacts, objects, or temporary restore schemas.

Health probes expose only stable component codes, timing, and status. Live
readiness reported Neon, private Blob, and Queue configuration ready while the
external runner remained truthfully `RUNNER_NOT_CONFIGURED`. This is dependency
proof, not a deployed availability or monitoring claim.

## Protected preview deployment regression

The first Vercel preview build exposed a contract defect: importing the default
case service resolved hosted configuration during Next.js page-data collection.
The build failed closed on missing build-time runtime values. A red subprocess
test reproduced the same import failure with preview mode and provider values
absent.

The fix wraps runtime selection in the same memoized lazy operations boundary.
Import and page-data collection no longer read credentials; the first actual
operation still validates once, preserves the error, and never falls back to
memory. The focused test passed 5/5, the complete local gate passed 172 Vitest
tests, and the six live provider tests remained green.

The exact fix commit then passed GitHub CI and produced a Ready, Vercel
SSO-protected preview. Authenticated probes returned liveness `200`, readiness
`200` with `DATABASE_READY`, `ARTIFACT_STORE_READY`, and
`QUEUE_CONFIGURATION_READY`, and runner `503` with the expected
`RUNNER_NOT_CONFIGURED`. This proves a protected development preview and its
durable dependencies, not stable hosting, public availability, deployed Queue
redelivery, an external runner, or an SLA. Deployment, project, resource,
request, and provider identifiers are omitted from committed evidence.

Tenant restore is transactionally atomic for Postgres state and uploads only
content-addressed objects that pass the manifest hashes before that transaction.
The live drill exported one complete verified tenant, removed its source Blob,
restored it into a fresh Neon schema, re-exported and digest-compared the
manifest, read the restored private bundle independently, and removed the
temporary schemas/object afterward. Logs did not contain the planted private
body marker.

No visual artifact applies to these backend foundation tasks; their observable
results are the migration ledger, database catalog, provider access behavior,
constraint behavior, restore digest, and command results. Provider resource
IDs, credentials, URLs, test tenant IDs, and object keys are omitted from the
committed evidence. Hosted browser and ChatGPT visuals remain required by the
later launch milestone.
