## RF-8103 proof boundary

PGlite runs real PostgreSQL in WebAssembly and closes the local SQL semantics
gate for schema and migration behavior. It does not claim a live Neon provider
round-trip. The real provider transaction/isolation evidence remains required
by `RF-8110` and the Milestone 8A acceptance gate.

The Neon adapter uses the current interactive-transaction `Pool` path, has
commit/rollback/release contract tests, and is created lazily so a
credential-free build performs no connection. These local adapter contracts do
not replace the live Neon concurrency/isolation gate.

The Vercel adapter always requests private immutable objects, uses
cache-bypassed reads, returns no provider URL, and prefers the platform's
short-lived OIDC identity over a static token. Local provider-contract tests do
not prove that an actual store rejects a direct unauthenticated URL; that live
private-store test remains required by the Milestone 8A acceptance gate.

The Vercel Queues adapter uses identifier-only messages, the durable event ID
as its provider idempotency key, bounded retention, the SDK's auto-renewing
push callback, and application-owned poison retry limits. Postgres remains the
authority for publisher claims, job leases, attempt bounds, terminality, and
recovery. The local adapter contract does not prove real publish/redelivery or
provider visibility behavior; those live Queue checks remain required by
`RF-8110` and the Milestone 8A provider gate.

Quota, cancellation, retention scheduling, deletion-claim recovery, and
append-only tombstone behavior are proven against PostgreSQL semantics. The
private-object deletion contract is proven with the same provider-neutral Blob
port used by the Vercel adapter, including provider failure and already-missing
object behavior. This is not a live Vercel Blob deletion claim; that provider
round-trip remains part of `RF-8110` and the Milestone 8A provider gate.

Health probes expose only stable component codes, timing, and status. The local
built-server smoke proves the route and failure-separation contract; it does
not claim live Neon, Vercel Blob, Vercel Queue, or runner availability. Those
provider checks remain part of `RF-8110` and the hosted launch gate.

No visual artifact applies to these backend foundation tasks; their observable
results are the migration ledger, database catalog, provider contracts,
constraint behavior, and command logs.
